# Bubble Tea v2 Migration

Scope: Migrate `flourish` from Bubble Tea/Bubbles/Lip Gloss v1 to v2 while preserving editor behavior (buffer semantics, wrap/layout correctness, cursor/selection, links, completion, snapshots), keeping custom highlighting, and removing editor-owned clipboard mechanics.

Documentation: `roadmap/v2.md`, `docs/editor.md`, `README.md`, `go.mod`, `editor/model.go`, `editor/update.go`, `editor/update_mouse.go`, `editor/completion_popup_render.go`, Bubble Tea v2 release notes (`https://github.com/charmbracelet/bubbletea/releases/tag/v2.0.0`), Bubbles v2 release notes (`https://github.com/charmbracelet/bubbles/releases/tag/v2.0.0`).

Legend: [ ] todo, [x] done.

## Phase 2: View API and Host Integration
- [ ] Switch editor view API to Bubble Tea v2 `tea.View` — Aligns with v2 rendering contract and view-level runtime flags.
  - Repository: `flourish`
  - Component: `editor`, examples
  - Scope: Change `editor.Model.View()` to return `tea.View` in `editor/model.go`; update examples in `examples/*` to return `tea.View` and compose editor content using `tea.NewView(...)`.
  - Snippets:
    ```go

    	return v
    }
    ```
  - Tests: update and run example tests plus package tests — expect unchanged runtime behavior with new return type.

## Phase 3: Input/Event Adaptation (Key + Mouse + Paste)
- [ ] Port key/mouse handling to v2 message model while preserving semantics — Keeps editing and navigation behavior stable under new event types.
  - Repository: `flourish`
  - Component: `editor` input pipeline
  - Scope:
    - Replace `tea.KeyMsg` branching with `tea.KeyPressMsg` (`editor/update.go`, completion key handling).
    - Replace v1 mouse action enum usage with v2 typed messages:
      - `tea.MouseClickMsg`
      - `tea.MouseMotionMsg`
      - `tea.MouseReleaseMsg`
      - `tea.MouseWheelMsg`
    - Keep existing selection drag and scrollbar interaction semantics.
  - Snippets:
    ```go
    switch msg := msg.(type) {
    case tea.KeyPressMsg:
    	// key handling
    case tea.MouseClickMsg, tea.MouseMotionMsg, tea.MouseReleaseMsg, tea.MouseWheelMsg:
    	// mouse handling
    }
    ```
  - Tests: migrate/update `editor/update_test.go`, `editor/scrollbar_mouse_test.go`, `editor/completion_*_test.go` — expect parity with pre-migration behavior.
